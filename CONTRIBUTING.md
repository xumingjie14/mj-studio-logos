# 贡献指南

欢迎为 MJ工作室 · Bloomberg Logos哨兵终端 项目做出贡献！

## 🎯 如何贡献

### 报告问题
如果你发现任何问题或有改进建议：
1. 在 Issues 页面查看是否已有类似问题
2. 如果没有，创建新的 Issue
3. 清晰描述问题，包括：
   - 问题描述
   - 复现步骤
   - 期望行为
   - 实际行为
   - 截图（如果适用）

### 提交代码
1. Fork 本仓库
2. 创建功能分支 (`git checkout -b feature/amazing-feature`)
3. 提交更改 (`git commit -m 'Add some amazing feature'`)
4. 推送到分支 (`git push origin feature/amazing-feature`)
5. 打开 Pull Request

## 📁 项目结构

了解项目结构有助于更好地贡献：

```
├── index.html              # 主界面文件
├── api/                    # Serverless Functions
│   └── liquidity.js       # 流动性数据API
├── .github/workflows/     # GitHub Actions
│   └── vercel-deploy.yml  # 自动部署配置
├── README.md              # 项目说明
├── CONTRIBUTING.md        # 贡献指南
└── LICENSE                # 许可证文件
```

## 🛠️ 开发环境设置

### 本地开发
```bash
# 克隆仓库
git clone https://github.com/your-username/mj-studio-logos.git

# 进入项目目录
cd mj-studio-logos

# 启动本地服务器
npx serve . -p 3000
```

### 代码规范
- 使用有意义的变量名和函数名
- 添加必要的注释
- 保持代码简洁
- 遵循现有代码风格

## 🧪 测试

### 手动测试清单
提交代码前请测试：
- [ ] 界面在不同浏览器中正常显示
- [ ] 响应式设计正常工作
- [ ] API端点返回正确数据
- [ ] 中文显示无乱码
- [ ] 时间更新功能正常

### 自动化测试
项目目前依赖手动测试，欢迎添加自动化测试！

## 📝 提交信息规范

使用清晰的提交信息：
```
feat: 添加新的监控面板
fix: 修复时间显示问题
docs: 更新README文档
style: 调整CSS样式
refactor: 重构API代码
test: 添加测试用例
chore: 更新依赖包
```

## 🔄 Pull Request 流程

1. **确保代码质量**
   - 代码通过基本测试
   - 遵循代码规范
   - 添加必要的注释

2. **描述更改**
   - 清晰说明更改内容
   - 说明为什么需要这个更改
   - 提供测试结果

3. **等待审查**
   - 维护者会审查代码
   - 可能需要一些修改
   - 通过后合并到主分支

## 🎨 设计贡献

### UI/UX 改进
- 保持彭博社终端风格
- 确保响应式设计
- 优化用户体验
- 添加必要的动画效果

### 数据可视化
- 使用清晰的图表
- 提供数据趋势分析
- 优化数据展示方式

## 📊 数据源贡献

### 添加新数据源
如果你想添加新的数据源：
1. 确认数据源的可靠性和稳定性
2. 确保有合法的API访问权限
3. 遵循项目的数据处理规范
4. 添加相应的错误处理

### 数据更新频率
- 实时数据：每分钟更新
- 日度数据：每天更新
- 历史数据：按需加载

## 🔐 安全注意事项

- 不要提交敏感信息（API密钥、密码等）
- 所有敏感信息通过环境变量管理
- 确保代码没有安全漏洞
- 遵循最小权限原则
